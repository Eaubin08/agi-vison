# Politique de Sécurité Obsidia AGI

## 1. La Sécurité par le Déterminisme
La sécurité d'Obsidia ne repose pas sur le secret, mais sur la **Vérifiabilité**. Le code est public, les preuves sont publiques, le Juge est public.

## 2. Signalement de Vulnérabilités
Si vous découvrez une dérive cognitive non interceptée par le Juge Déterministe, veuillez nous contacter immédiatement. Une dérive est une faille critique.

## 3. Audit Ex-Ante
Chaque mise à jour du système est soumise à un audit ex-ante via le standard X-108. Aucune modification ne peut être scellée sans une preuve de conformité mathématique.
